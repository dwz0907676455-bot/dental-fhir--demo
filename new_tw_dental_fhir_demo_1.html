臺中陽光牙醫為病人 王小明 進行全口檢查，建立以下 FHIR 資源：

步驟 2：上傳至共享 FHIR Server
A 診所將 Bundle（含 Patient、Encounter、Observation、DiagnosticReport）POST 至衛福部 NHIS FHIR Server。
POST https://fhir.nhis.gov.tw/R4/Bundle Content-Type: application/fhir+json Authorization: Bearer <A診所系統Token> { "resourceType": "Bundle", "type": "transaction", "entry": [ { "resource": { "resourceType": "Organization", "id": "ORG-109890" } }, { "resource": { "resourceType": "Patient", "id": "PID001" } }, { "resource": { "resourceType": "Encounter", "id": "enc-20260310" } }, { "resource": { "resourceType": "Observation", "id": "obs-26" } }, { "resource": { "resourceType": "Observation", "id": "obs-36" } }, { "resource": { "resourceType": "DiagnosticReport", "id": "dr-001", "result": [ { "reference": "Observation/obs-26" }, { "reference": "Observation/obs-36" } ] }} ] } HTTP 200 OK ─ Bundle 已接受並儲存至 NHIS FHIR Server

步驟 4：B 醫院查閱病人歷史
台南成大附屬醫院醫師，以病人 PID 查詢 FHIR Server，取得 A 診所的全部牙科紀錄。

FHIR 資源層次說明
Patient（長期累積）
病人主體，跨院共用同一 PID，所有就診紀錄的根節點。
Encounter（每次就診一筆）
每次到診所就診建立一個 Encounter，記錄日期、診所、醫師。Observation 掛在 Encounter 下。
Observation（每顆牙一筆）
單顆牙齒狀態，以 FDI extension 標記牙位，可查詢同一顆牙的跨次就診歷史。
DiagnosticReport（參考 Observation）
X 光或 AI 判讀報告，result 欄位 reference 到相關 Observation，不重複儲存單顆牙資料。
Consent（病人授權）
病人首次至新醫院就診時，授權該機構調閱全部（或特定專科）的歷史與未來紀錄。可隨時撤銷。
查詢範例
// Q2: 查一顆牙的歷史 GET /Observation?patient=PID001 &extension=https://twdental.org/fhir/ StructureDefinition/tooth-fdi-number|26 &_sort=-date // Q3: 查病人所有就診 Encounter GET /Encounter?patient=PID001&_sort=-date // Q3: 查特定 Encounter 的所有 Observation GET /Observation?encounter=enc-20260310 // Q4: 查有 DiagnosticReport 的 Observation GET /DiagnosticReport?patient=PID001 &category=dental // 授權後跨院全量查詢 GET /Patient/PID001/$everything
資料安全架構
🔐
SMART on FHIR OAuth 2.0
病人可透過 SMART on FHIR 授權機制查看個人資料，Access Token 15 分鐘短效。牙醫師以執照身份即可調閱病人跨院紀錄，無需病人每次個別授權。
🔒
TLS 1.3 全程加密
所有 FHIR API 傳輸均經 TLS 1.3 加密，EV SSL 憑證每 90 天自動更新。
🛡
Patient ID 與身分資料分離
系統僅流通 PID，身分證號加密儲存於獨立資料庫。
🔍
AuditEvent 稽核
每筆資料讀寫均產生 FHIR AuditEvent，保存 7 年。
🏥
跨院資料互通
每間診所有唯一 ORG ID，持有合法執照的牙醫師可直接查詢病人在任何診所的歷史 Observation，不需病人額外授權，實現真正的跨院資料共享。
📋
Consent 病人授權（新增）
病人首次至特定醫院就診，可授權該醫院調閱全部或特定專科（如牙科）的過去及未來病歷。授權可隨時由病人自行撤銷。
💾
localStorage 本地持久化（Demo）
Demo 環境使用 localStorage 持久化，關閉瀏覽器資料仍保留。
